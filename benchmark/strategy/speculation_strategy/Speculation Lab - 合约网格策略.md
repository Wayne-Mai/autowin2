# 合约网格策略 — Speculation Lab (OKX)

## TL;DR
合约网格解决了现货网格只多不空、灵活性差、资金利用率低的问题。可根据市场调整多空倾向（做多/做空/中性三种属性），结合杠杆放大效果。视频示例 BTC/USDT 22500-25000，10 等差网格，5 倍杠杆，1000U 保证金。

## Setup / 形态
- 适用市场：震荡行情（震荡上行→做多网格；震荡下行→做空网格；中性→市价上方开空、下方开多）
- 平台：OKX 策略广场 → 网格策略 → 合约网格
- 三种属性：做多 / 做空 / 中性
- 网格类型：等差 / 等比

## Entry Rules / 入场（4 步走）

**做多网格示例（02:42–04:55）**
1. **明确网格属性**：选择做多网格
2. **确定价格区间**：最低价 22500，最高价 25000
3. **模式和数量**：等差，10 个网格
   - 系统自动算单网格利润 4.33%-4.83%（已扣手续费）
4. **杠杆和金额**：5 倍杠杆，1000U 保证金 → 网格总价值 5000U

**高级设置**：
- 触发条件：默认（也可设价格触发：标记一个价格等待成交）
- 止盈：突破区间到合适位置（视频例 25300）
- 止损：跌破震荡区间合理位置（视频例 22000）

**做空网格示例（05:05–06:03）**
- 步骤相同，但方向相反
- 触发条件可设在 24800
- 止盈 22000，止损 25300
- 入场停止条件选择手动停止

**中性网格**：
- 在策略开启时的市场价格：上方开空平空、下方开多平多

## Exit & Stop Loss / 出场止损
- 止盈：突破区间到合适位置，仓位获利平
- 止损：跌破/突破网格区间合理位置，策略停止
- 合约不同于现货 → 有爆仓风险，加杠杆放大损失，**必须**设止损
- 移仓 / 关键风险提示：见 Caveats

## Risk Management / 仓位
**三件必注意（06:23–07:35）**：
1. **网格并不是越多越好** —— 涉及手续费
   - 单网格收益必须 > 2 × 手续费
   - 平台手续费 0.05%，一买一卖 0.1% → 利润少于 0.1% 就亏损
2. **设震荡区间时尽量选大级别支撑阻力位置**，并为异常波动准备
   - 设好止损或行情警报
   - 有应对风险的预案
3. **大部分资金配置到现货，只用小部分资金做合约网格**
   - 真正赚大钱的机会在小部分趋势行情中
   - 网格交易当做"仓位策略"，目的不在高回报而在降低风险

## Examples / 实例
- BTC/USDT，22500-25000，等差 10 网格，5 倍杠杆，1000U 保证金
- 单网格利润 4.33%-4.83%
- 做多：止盈 25300，止损 22000
- 做空：止盈 22000，止损 25300，触发条件 24800

## Caveats / 注意
- 合约带杠杆 → 一定要设止损
- 跌破/突破区间后策略不再运行，但仓位还在 → 不设止损会持续亏损
- 视频强调"渔夫打渔" — 一网撒下去后，只要价格在网里就有利可图；破网后价格一去不返也不会继续操作
- "交易市场绝大部分时间都是处在起伏不定的震荡走势，而真正赚大钱的机会都是在那小部分趋势行情中"
