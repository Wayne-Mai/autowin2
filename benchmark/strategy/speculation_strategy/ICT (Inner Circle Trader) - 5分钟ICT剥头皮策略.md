# 5分钟ICT剥头皮策略 — ICT (Inner Circle Trader)

## TL;DR
ICT 本人在 Ins 公开的5步法 ICT 加密策略：(1) 标记四个高波动时段；(2) 30分钟图找流动性区域；(3) 5分钟图等趋势结构破坏；(4) 找到FVG；(5) 在FVG缺口的上沿/中间入场，止损放缺口外，止盈固定比例或前高前低 (00:40-05:30)。

## Setup / 形态
**第一步：标记适合交易的重要时段** (00:40-02:32)

加密货币24小时不间断交易，大部分是无序波动时间。四个高波动性时段 (北京时间)：

### 时段1：早 7:00-9:00 (00:53-01:33)
- 世界通用的8点钟日线收盘前后
- 多空经常会在此集中博弈
- 大部分交易员集中在亚洲地区，刚睡醒易情绪化
- 大资金会在此时间带动散户操作，制造追涨杀跌
- 容易出现当日高点或低点

### 时段2：凌晨 3:00-5:00 (01:33-01:58)
- 当日4小时K线第一次收盘前后
- 亚洲地区交易员基本休息
- 大资金机构选择大多数人不方便加保证金的时间，集中打掉止损挂单或直接把资金打到爆仓

### 时段3：凌晨 23:00-1:00 (01:58-02:19)
- 日内交易员休息前决定是否平仓
- 大资金常在此时段做盘画线，等到 3-5 点钟收割
- 大幅波动后此时段经常冲高回落 → 可短线交易

### 时段4：下午 4:00-6:00 (02:19-02:30)
- 美欧交易员早盘时间
- 由于美欧交易员数量和资金比例有限，行情波动经常不太明显

## Entry Rules / 入场

### 第二步：30分钟图标记流动性区域 (02:30-02:54)
- 在交易时段之前打开30分钟图表
- 找到近期波段高/低点用**直线**标记出来
- 这就是**流动性区域** = 市场上有大量未执行买/卖单的价格位置
- 通常是买卖成交的热点
- 等待价格到达流动性区域附近去消耗其流动性

### 第三步：切换5分钟等待结构转变 (02:54-03:19)
- 切换到5分钟图表
- 以多头为例：价格持续下行到流动性区域后，等价格止跌回升直接突破前一个波段的高点
- 这表示下跌趋势的"更低高点+更低低点"结构已被破坏
- 价格很可能由空变多

### 第四步：找到公允价值缺口 FVG (03:19-03:50)
- 加密货币市场基本只用第一种FVG类型
- FVG形态由 **3 根 K 线**组成
- 中间K线的实体与相邻K线的影线不重叠的部分 = 缺口
- 缺口表示市场在该价格区间内的交易活动突然减少，形成单方向缺口
- 缺口可能在未来某个时间被填补
- 填补完成后较大概率继续沿趋势方向运行

### 第五步：进出场位置 (03:50-04:13)
**多头**：
- 当缺口价格幅度较大且价格再次回到缺口区域时
- 在**缺口的上沿或中间区域**入场多单
- 止损设置在 **FVG 缺口下方或前低附近**
- 止盈选择**固定比例或前高附近**

**空头规则** (04:13-05:07)：
- 时段过滤同上 (例：早 7-9点)
- 30分钟图找波段高低点
- 切到5分钟图，等价格上行到流动性区域，遇阻回落，跌破前一个波段的低点
- 这表示上涨趋势的"更高高点+更高低点"结构被破坏
- 用矩形工具画出 FVG
- 当价格再次反弹到 FVG 缺口区域时
- 在**缺口的上沿或中间区域**入场空单
- 止损设置在 **FVG 缺口上方或前高附近**
- 止盈**固定比例或前低位置**

## Exit & Stop Loss / 出场止损
**止损**：FVG缺口对侧 (多 = 缺口下方/前低；空 = 缺口上方/前高)

**止盈**：
- 固定比例
- 或前高/前低位置

## Examples / 实例
**比特币多头例子** (02:54-04:13)：
- 标记早 7-9点钟时段
- 30分钟图标记近期波段高低点
- 切5分钟，等价格持续下行到流动性区域
- 等价格止跌回升突破前一波段高点 (结构转变)
- 找到由3根K线组成的FVG
- 在FVG上沿/中间入场多单，止损放缺口下方

**空头例子** (04:13-05:07)：
- 标记早 7-9点钟时段
- 30分钟图找波段高低点
- 价格持续上行到流动性区域后遇阻回落，跌破前一波段低点
- 画FVG，回测FVG时入场空单，止损放缺口上方
- "这笔交易很快就完成了止盈"

## Caveats / 注意
- 视频强调一刻不离屏幕是不好的交易状态 → 应等候四个高波动时段以外其他时段不宜大量交易
- 加密货币市场基本只用第一种FVG类型 (三K线模型)
- ICT主张通过识别流动性、订单区块、FVG等区域，结合多时间框架分析和风险管理 (05:07-05:21)
- 用这个交易框架再结合其他指标/形态/策略 → 形成属于自己的交易系统
