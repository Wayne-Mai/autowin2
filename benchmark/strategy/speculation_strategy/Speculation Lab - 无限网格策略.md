# 无限网格交易策略 — Speculation Lab

## TL;DR
无限网格 = 网格交易的"飞升版本"，特点是**有下限，无上限**。通过持续高抛低吸，在上涨行情中始终保持与"头寸价值"等值的货币对（如始终保持 500U 价值的 BTC，多于则卖出，少于则补齐）。适合震荡上行的慢牛市，只能做多不能做空。

## Setup / 形态
- 适用市场：震荡上行的慢牛市，大市值币种（BTC 优先；视频也提到 BTC + LTC + ETH 三架马车组合）
- 不适用：大级别下跌趋势、暴涨暴跌币种（会触发大量买卖增加手续费）
- 操作平台：OKX 策略广场 → 网格策略 → 无限网格

## Entry Rules / 入场
**核心机制（00:38–01:43）**
- 假设本金 1000U，在 25,000 美金价格用 500U 买入 BTC，每个网格利润率设 1%
- 价格上涨时：在 25,250 / 25,502.5 等等价位阶段止盈（按等比卖出）。每次卖出后剩余 BTC 兑换 USDT 仍能换回 500U。
- 价格下跌时：在 24,750 / 24,502.5 等等价位阶段买入。每次买入后 BTC 价值仍为 500U。
- 仓位价值始终保持在头寸值（500U），多了卖出，少了补齐。直到最小精度。

**OKX 创建步骤（02:19–05:09）**
1. 打开 OKX 策略广场 → 网格策略 → 无限网格 → 创建策略
2. 可选"推荐策略"（OKX 智能算法回测近 30 天） 或"自定义参数"
3. **确定价格区间最低价**：尽量放在大级别需求区或支撑位附近。跌破最低价后策略停止运行，但仓位会承受浮亏。
4. **网格收益率**：示例设为 1%。等比公式：每档价格 = 上一档价格 × (1 + 单网格利润率)
5. **入场价**：示例限价 27,000 入场。卖出网格自动算在 27,270 / 27,542.7 等；买入网格自动算在 26,732.7 / 26,468 等。
6. **投入金额**：示例 1000U
7. **触发类型**：立即触发（市价）/ 价格触发（限价挂单）/ RSI 触发（不常用）
8. **止盈**：在大周期阻力或供应区位置，如 29,500U。不设则持续套利。
9. **止损**：必须小于区间最低价，比如 23,900U（在大周期支撑或需求区附近）

## Exit & Stop Loss / 出场止损
- 止盈：上方大周期阻力/供应区
- 止损：必须 < 区间最低价；放在大周期支撑/需求区附近
- 跌破最低价后策略不再运行，但仍持有头寸等值仓位 → 必须设止损作为安全线
- 设了止盈达到价格后仓位会被全部卖出

## Risk Management / 仓位
五条经验（05:09–07:25）：

1. **判断市场条件**：只用在震荡慢牛市；大下跌或暴涨暴跌都会产生大量交易拉高成本。只选大市值币种（如 BTC）。
2. **网格间距**：根据历史波动幅度。波动大币种间距大；间距宽 → 交易少，需更大波动获利；间距窄 → 单笔利润小，手续费拖累严重。
3. **仓位管理**：使用较小头寸。等比运行下，未结头寸数量可能随网格扩大而增加。小仓位保护账户不在极端行情大幅缩水。
4. **风险管理**：必须设止损（网格下沿合适位置）。如果价格持续下跌出区间且不返回，账户承受较大浮亏。
5. **监控和调整**：密切关注市场和未结头寸。市场条件大变可手动停止策略。注意网格资金从交易账户隔离锁定，若有合约订单要留意保证金充足。

## Examples / 实例
- BTC/USDT，入场 27,000，网格收益率 1%，投入 1000U，止盈 29,500，止损 23,900
- 推荐组合："比特币 + 莱特币 + 大姨太"三架马车

## Caveats / 注意
- 只能做多，不能做空
- 不能用在持续下跌或波动剧烈的币
- 每次运行资金量较小但占用资金较多，适合需要拉长获利区间的朋友
- 必须提前在历史走势进行复盘
- 调好网格间距、头寸、监测风险
